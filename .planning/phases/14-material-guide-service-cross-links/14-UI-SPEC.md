## Component Inventory

### New Component: MaterialServiceCTA

| Property | Value |
|----------|-------|
| File path | `src/components/sections/material-service-cta.tsx` |
| Type | Server Component (no "use client") |
| Pattern source | ProblemSolutionCTA (src/components/sections/problem-solution-cta.tsx) |
| Semantic element | `<div>` within a `<section>` (SectionWrapper provides the section) |

**Props interface:**

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| materialName | string | yes | Display name of the material (e.g. "Asphalt Shingles") |
| serviceLinks | { title: string; path: string }[] | yes | Array from getMaterialRelatedServices() |

**Empty state:** Component returns null when serviceLinks.length === 0. No empty state UI rendered.

**Visual specification:**

- Outer container: `bg-secondary rounded-lg p-8 border border-accent` (matches ProblemSolutionCTA exactly)
- Heading: `text-[1.75rem] font-heading font-bold text-text-primary` -- reads "Professional {materialName} Services"
- Narrative paragraph: `text-lg text-text-secondary mt-4 leading-relaxed` -- reads "Our team specializes in {materialName} installation, repair, and maintenance across Hudson County. Explore our related services:"
- Service link pills container: `flex flex-wrap gap-3 mt-6`
- Each service link pill: `inline-flex items-center gap-2 bg-accent text-dominant font-bold px-4 py-2 rounded-lg min-h-[44px] hover:bg-accent-hover transition-colors duration-[150ms]`
- ArrowRight icon (16px) inside each pill, `aria-hidden="true"`
- Phone CTA below pills: `flex items-center gap-2 mt-4 text-accent text-lg font-bold`
- Phone icon (20px), `aria-hidden="true"`
- Phone number from BUSINESS_INFO.phone

**Accessibility:**
- All links are semantic `<Link>` (next/link) elements with visible text
- Phone link is semantic `<a href="tel:...">` with visible phone number text
- Touch targets meet 44px minimum height via min-h-[44px]
- Focus ring inherited from global `.focus-ring` class (not needed -- default browser focus ring suffices for links)
- ArrowRight and Phone icons use `aria-hidden="true"` to prevent screen reader duplication

### Modified File: Material Guide Page Template

| Property | Value |
|----------|-------|
| File path | `src/app/(marketing)/guides/materials/[slug]/page.tsx` |
| Change type | Add import, add function call, add JSX section |

**Insertion point:** Between section 13 (FAQ Accordion, tone="dominant") and section 14 (Related Guides, tone="secondary"). The new section uses tone="dominant" to maintain the alternating tone pattern:

| # | Section | Tone |
|---|---------|------|
| 13 | FAQ Accordion | dominant |
| **13.5** | **MaterialServiceCTA (NEW)** | **secondary** |
| 14 | Related Guides | secondary |

Wait -- checking the tone alternation: FAQ is dominant (13), Related Guides is secondary (14). Inserting a section between them at tone="secondary" would create two consecutive secondary sections. To maintain alternation, the new section should use tone="dominant" and Related Guides stays secondary. However, the ProblemSolutionCTA on problem pages uses a dominant SectionWrapper with a bg-secondary inner card. The card itself provides the visual contrast. This matches the existing pattern:

**Correct tone assignment:**

| # | Section | Tone | Rationale |
|---|---------|------|-----------|
| 13 | FAQ Accordion | dominant | Existing |
| **13.5** | **MaterialServiceCTA (NEW)** | **secondary** | The bg-secondary card on a secondary background creates a uniform panel look. This matches how ProblemSolutionCTA renders on problem pages (dominant wrapper + secondary card). However, to avoid visual confusion, use tone="dominant" so the secondary card contrasts against the dominant background -- exactly matching the ProblemSolutionCTA rendering on problem pages. |
| 14 | Related Guides | secondary | Existing |

**Final decision:** Use `tone="dominant"` for the MaterialServiceCTA SectionWrapper. The bg-secondary card sits on the dominant background, creating visual contrast. Related Guides then follows with tone="secondary" as it already does. This creates: dominant (FAQ) -> dominant (MaterialServiceCTA) -> secondary (Related Guides). Two consecutive dominant sections is acceptable because the MaterialServiceCTA card (bg-secondary with border-accent) provides its own visual distinction from the plain dominant content above it.

**Conditional rendering:** Wrap in `{serviceLinks.length > 0 && (...)}` guard, matching the Related Guides pattern on line 237 of the current page.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Service link hover | Background transitions from #c89640 to #d8aa56 over 150ms (transition-colors duration-[150ms]) |
| Service link click | Navigates to service page via next/link client-side navigation |
| Phone link hover | Opacity reduces to 80% (hover:opacity-80 transition-opacity) -- wait, ProblemSolutionCTA does NOT have opacity hover on phone. It has no explicit hover. Use plain text-accent with no hover effect, matching ProblemSolutionCTA exactly. |
| Phone link click | Opens device phone dialer via tel: href |
| ScrollReveal | Fade-in on scroll intersection (existing ScrollReveal wrapper, no new animation logic) |
| Keyboard navigation | Tab focuses each link sequentially. Enter/Space activates. Default browser focus ring. |

## Layout Contract

**MaterialServiceCTA card layout (inside SectionWrapper max-w-[1280px]):**

```
+---------------------------------------------------------------+
| SectionWrapper tone="dominant" (bg-dominant, py-12/16/20)     |
|  +----------------------------------------------------------+ |
|  | Card: bg-secondary rounded-lg p-8 border border-accent   | |
|  |                                                          | |
|  | [H2] Professional {materialName} Services                | |
|  |                                                          | |
|  | [P] Our team specializes in...                           | |
|  |                                                          | |
|  | [Pill] Roof Replacement ->  [Pill] Roof Repair ->        | |
|  |                                                          | |
|  | [Phone] (201) 555-0123                                   | |
|  +----------------------------------------------------------+ |
+---------------------------------------------------------------+
```

**Responsive behavior:**
- Mobile (< 640px): Full-width card with p-8. Service link pills stack naturally via flex-wrap. All text left-aligned.
- Tablet (640px-1024px): Same layout, more horizontal space for pills to sit side-by-side.
- Desktop (> 1024px): Same layout within max-w-[1280px] container. Pills always fit on one line (2-3 pills max per material).

No breakpoint-specific overrides needed -- the flex-wrap pattern handles all viewport sizes.
