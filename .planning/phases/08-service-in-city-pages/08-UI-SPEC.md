## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge pill `py-1` |
| sm | 8px | Compact element spacing, card gap within groups |
| md | 16px | Default element spacing, section inner padding, grid gaps |
| lg | 24px | Section padding, card `p-6`, gap between major elements |
| xl | 32px | Layout column gaps (`lg:gap-8`) |
| 2xl | 48px | Section vertical padding (mapped to `py-12`) |
| 3xl | 64px | Section vertical padding on large screens (mapped to `py-16` / `py-20`) |

Exceptions:
- Touch targets: all interactive elements use `min-h-[44px] min-w-[44px]` per WCAG 2.2 Target Size
- Hero section vertical padding: `py-16 sm:py-20 lg:py-24` (64px / 80px / 96px) -- inherited from ServiceHero and CityHubHero patterns
- Max content width: 1280px (`max-w-[1280px]`) with responsive horizontal padding `px-4 sm:px-6 lg:px-8`

Source: SectionWrapper, ServiceHero, CityHubHero -- established Phase 2/5/7 patterns

## Interaction Contracts

### Scroll-Linked Animations
All sections (except hero, BadgeStrip, MidPageCTA, TestimonialCarousel, QuoteForm, CTABanner) wrapped in `<ScrollReveal>` for fade-in-on-scroll using Motion library. Respects `prefers-reduced-motion: reduce` via globals.css media query.

### FAQ Accordion
- `"use client"` component using `useState` + `AnimatePresence`
- First FAQ open by default (`defaultOpenIndex={0}`)
- Single-expand behavior (opening one closes others)
- ChevronDown icon rotates 180deg on expand
- Expand/collapse animation: height 0 to auto, opacity 0 to 1, duration 300ms, ease "easeOut"
- ARIA: `aria-expanded`, `aria-controls`, `role="region"`, `aria-labelledby`

### Material Cards
- `"use client"` component with expand/collapse for pros and cons
- Multi-expand behavior (each card independent)
- Toggle link: `text-accent text-sm underline` with `aria-expanded`

### Testimonial Carousel
- Embla Carousel with `loop: true`, `align: "start"`, autoplay every 5000ms
- Mobile: 1 slide visible. Desktop: 3 slides visible (`lg:flex-[0_0_33.333%]`)
- Previous/Next buttons: 44px circular touch targets with ChevronLeft/ChevronRight
- ARIA: `aria-roledescription="carousel"`, `aria-label`, each slide has `role="group" aria-roledescription="slide"`

### Internal Navigation Links
- Service cards in SiblingCitiesNav: `hover:border-accent transition-colors duration-[150ms]`
- Current city: visually distinct `border-accent text-accent font-bold`, not clickable
- All internal links use `next/link` with `prefetch={false}` (96 pages, avoid prefetch overhead)

### CompactQuoteForm (in hero)
- Pre-fills service type from page context via `defaultServiceType` prop
- Form validation via Zod + React Hook Form (established Phase 3 pattern)
- Turnstile invisible captcha

## Accessibility Checklist

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<section>`, `<nav>`, `<h1>`-`<h3>` strict hierarchy, `<article>` not needed (not blog) |
| Single H1 | CityServiceHero renders the only `<h1>` per page |
| Heading hierarchy | H1 (hero) > H2 (section headings) > H3 (card headings, neighborhood names) -- no skipping |
| Keyboard navigation | All links, buttons, accordion triggers reachable via Tab; `focus-visible` ring on all |
| Focus ring | `box-shadow: 0 0 0 2px var(--color-dominant), 0 0 0 4px var(--color-accent)` via `.focus-ring` class |
| Touch targets | `min-h-[44px] min-w-[44px]` on all interactive elements |
| Color contrast | Text primary #f0ede6 on dominant #2a2e22 = 10.7:1 (AAA). Text secondary #b0ae9e on dominant #2a2e22 = 5.8:1 (AA). Accent #c89640 on dominant #2a2e22 = 4.5:1 (AA). |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all animations in globals.css |
| ARIA patterns | Accordion: `aria-expanded`, `aria-controls`. Carousel: `aria-roledescription`. Nav: `aria-current="page"`, `aria-label`. |
| Image alt text | No images in this phase (text-only content pages). If images added later, require descriptive alt text per SEO-12. |
| Screen reader | SiblingCitiesNav uses `<nav aria-label="...">`. Current city uses `aria-current="page"`. |

Source: globals.css `.focus-ring` class, WCAG AA requirements (UX-05, UX-06), established patterns in FaqAccordion and TestimonialCarousel
