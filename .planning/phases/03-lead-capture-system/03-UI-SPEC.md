## Spacing Scale

Declared values (must be multiples of 4). Inherited from Phase 2 established patterns.

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4px | Gap between required-field asterisk and label text, inline icon gaps |
| sm | 8px | Space between error message and input field, gap between form fields in compact form |
| md | 16px | Default gap between form fields in full form, inner padding of form container |
| lg | 24px | Section padding (horizontal), gap between form heading and first field |
| xl | 32px | Form container vertical padding, gap between form section and surrounding content |
| 2xl | 48px | Section vertical padding (mobile: py-12 = 48px) |
| 3xl | 64px | Section vertical padding (desktop: py-16 to py-20 range) |

Exceptions:
- Touch targets: 44px minimum height and width on all interactive elements (buttons, inputs, selects). Established in Phase 2 button-variants.ts via `min-h-[44px] min-w-[44px]`.
- Form input height: 48px (py-3 at 18px font size achieves this). Provides comfortable touch target on mobile.

Source: Phase 2 SectionWrapper (py-12/sm:py-16/lg:py-20), button-variants.ts (min-h-[44px])

## Form Input Visual Contract

All form inputs (FormInput, FormSelect, FormTextarea) share these visual properties via cva variants.

### Default State
- Background: `bg-dominant` (#2a2e22)
- Border: 1px solid `border-secondary/50` (semi-transparent #3e4435)
- Border radius: `rounded-md` (6px)
- Text color: `text-text-primary` (#f0ede6)
- Placeholder color: `text-text-secondary/60` (#b0ae9e at 60% opacity)
- Padding: `px-4 py-3` (16px horizontal, 12px vertical) for default size; `px-3 py-2` for compact size
- Font: 18px Cormorant Garamond, weight 500
- Transition: color change at 150ms (`duration-[--duration-fast]`)

### Hover State
- Border: 1px solid `border-accent/50` (semi-transparent #c89640)

### Focus State
- Border: 1px solid `border-accent` (#c89640)
- Ring: 2px `ring-accent` with 1px offset in dominant color
- Outline: none

### Error State
- Border: 1px solid `border-destructive` (#ef4444)
- Ring on focus: 2px `ring-destructive` (#ef4444) replacing accent ring
- Error message below: 14px, `text-destructive` (#ef4444), `role="alert"`, linked via `aria-describedby`

### Disabled State
- Opacity: 50%
- Cursor: not-allowed

### Select-Specific
- Custom dropdown arrow icon using Lucide `ChevronDown` (16px, text-secondary)
- Appearance: none (removes native select arrow)
- Same dimensions and styling as FormInput

### Textarea-Specific
- Minimum height: 120px (5 rows)
- Resize: vertical only
- Same border/focus/error styling as FormInput

## Form Layout Contract

### Compact Quote Form (3-field)

**Placement:** Near hero section, wrapped in SectionWrapper with `id="quote-form"`. This is the scroll target for header CTA.

**Layout:**
- Desktop (lg+): Single horizontal row with 3 fields side-by-side + submit button. Uses CSS Grid: `grid-cols-[1fr_1fr_1fr_auto]` with `gap-3` (12px).
- Tablet (sm-lg): 2x2 grid. Row 1: name + phone. Row 2: service type + submit button. `grid-cols-2 gap-3`.
- Mobile (<sm): Single column stack. Each field full width, submit button full width. `grid-cols-1 gap-3`.

**Visual container:**
- Background: `bg-secondary` (#3e4435) -- stands out from dominant page background
- Border radius: `rounded-lg` (8px)
- Padding: `p-6` (24px) on desktop, `p-4` (16px) on mobile
- No visible border (contrast with background provides separation)

**Heading:** "Get Your Free Quote" in 28px Cormorant bold, above the form fields.
**Subtext:** "Quick response -- we'll call you back within 24 hours." in 18px text-secondary, below heading.

**Submit button:** Full accent primary style, text "Get Free Quote", compact size in horizontal layout, full-width on mobile stack.

### Full Quote Form (6-field)

**Placement:** Near bottom of page content, before footer. Wrapped in SectionWrapper with `id="full-quote-form"`.

**Layout:**
- Desktop (lg+): 2-column grid for short fields, full-width for textarea. Row 1: name + phone. Row 2: email + service type. Row 3: address (full width). Row 4: message textarea (full width). Row 5: submit button (left-aligned). `grid-cols-2 gap-4` with `col-span-2` for full-width fields.
- Tablet (sm-lg): Same 2-column layout as desktop.
- Mobile (<sm): Single column stack. All fields full width. `grid-cols-1 gap-4`.

**Visual container:**
- Background: `bg-secondary` (#3e4435)
- Border radius: `rounded-lg` (8px)
- Padding: `p-8` (32px) on desktop, `p-6` (24px) on tablet, `p-4` (16px) on mobile
- No visible border

**Heading:** "Request a Detailed Quote" in 28px Cormorant bold, above the form fields.
**Subtext:** "Tell us about your project and we'll prepare a comprehensive estimate." in 18px text-secondary, below heading.

**Submit button:** Full accent primary style, text "Request Free Quote", default size, left-aligned (not centered, not full-width on desktop).

### Contact Page Form

Uses the same QuoteForm (6-field) component with identical visual contract. The contact page provides the surrounding context (heading, NAP info, map) -- the form component itself is reused without modification. No `defaultServiceType` prop is passed on the contact page (dropdown shows placeholder).

## Interaction States Contract

### Form Submission Flow

**1. Idle State:**
- All fields empty (or with defaults). Submit button enabled, accent background.
- Turnstile widget initializing invisibly in the background.

**2. Validation on Blur:**
- When user leaves a required field empty or with invalid data, error message fades in below the field.
- Animation: `AnimatePresence` with `motion.p` -- fade in from 0 opacity + slide down 4px, duration 200ms.
- Field border transitions from default to destructive.
- Error message uses `role="alert"` for screen reader announcement.

**3. Submitting State:**
- Submit button: text replaced with Lucide `Loader2` icon (20px) spinning animation + "Submitting..." text. Button background remains accent but at 80% opacity. Button is disabled (`pointer-events-none`).
- All form fields become disabled (50% opacity, not-allowed cursor).
- No full-page overlay or modal.

**4. Success State:**
- Entire form (fields + button) fades out and is replaced by success message in-place.
- Animation: form exits with fade-out (150ms), success message enters with fade-in + scale from 0.95 to 1.0 (300ms). Uses `AnimatePresence` with `mode="wait"`.
- Success container: `bg-secondary` with `border-l-4 border-success` left accent bar. Padding `p-6`.
- Icon: Lucide `CheckCircle` (32px, `text-success`) centered above text.
- Heading: "Thank You!" in 28px Cormorant bold, text-primary.
- Body: "We've received your request and will contact you within 24 hours." in 18px text-primary.
- Phone fallback: "Need immediate help? Call us at [(201) 555-0123](tel:+12015550123)" in 18px text-secondary. Phone number is accent-colored `tel:` link.

**5. Error State:**
- Form fields remain visible and editable (data preserved per D-10).
- Error banner slides in above the form.
- Animation: `motion.div` with slide-down from -16px + fade-in, duration 300ms.
- Error container: `bg-destructive/10` (10% red tint) with `border-l-4 border-destructive` left accent bar. Padding `p-4`. Rounded-md.
- Icon: Lucide `AlertCircle` (20px, `text-destructive`) inline with text.
- Copy: "Something went wrong. Please try again or call us at [(201) 555-0123](tel:+12015550123)." in 18px text-primary. Phone number is accent-colored `tel:` link.
- Submit button returns to idle state (re-enabled, accent background).
- Turnstile widget resets automatically to obtain fresh token.

### Scroll-to-Form Behavior

**Header CTA ("Get Free Quote" button):**
- On click: check if `#quote-form` element exists in DOM.
- If exists: `scrollIntoView({ behavior: "smooth", block: "start" })` with offset for sticky header height (80px default, 56px scrolled).
- If not exists: navigate to `/contact` page.
- Implementation: change header CTA from `<Link href="/contact">` to `<button>` with onClick handler. Preserves all existing visual styling via `buttonVariants`.

**In-page CTA buttons:**
- CTA buttons within page content that say "Get Free Quote" or similar anchor-link to `#quote-form` (compact form at top).
- CTA buttons that say "Request Detailed Quote" anchor-link to `#full-quote-form` (full form near bottom).
- Use same smooth scroll behavior with header offset.

### Service Type Auto-Selection

- On service pages (e.g., `/services/residential/roof-repair/`): the service type dropdown pre-selects the current service name. Passed as `defaultServiceType` prop from the page component.
- On non-service pages (homepage, contact, etc.): dropdown shows placeholder text "Select a service..." with empty string value.
- User can always change the selection regardless of pre-selection.
- The placeholder option is styled in `text-text-secondary/60` to differentiate from real selections.

### Honeypot Field

- Hidden from all users (visual + assistive technology).
- Uses: `aria-hidden="true"`, `tabIndex={-1}`, positioned absolutely off-screen (`absolute -left-[9999px] -top-[9999px] h-0 w-0 overflow-hidden`).
- Field name: "website" (bots commonly fill URL fields).
- No label visible. `autoComplete="off"`.
- Zod schema: `z.string().max(0).optional()` -- any content means bot.

### Turnstile Widget

- Invisible mode (`appearance: "interaction-only"`): no visual presence unless Cloudflare needs to challenge.
- Widget container: zero-height div within the form, no visible footprint.
- Token obtained silently before form submission.
- On token expiry (300s): widget resets automatically via `expired-callback`, new token obtained.
- After each submission (success or failure): widget reset to obtain fresh single-use token.
- Test keys used in development: `1x00000000000000000000AA` (always pass).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Every input has a visible `<label>` linked via `htmlFor`/`id` |
| Required fields | `required` attribute on HTML element + visual asterisk (`aria-hidden="true"` on asterisk to avoid "star" announcement) |
| Error association | `aria-invalid="true"` on invalid fields + `aria-describedby` pointing to error message element |
| Error announcement | Error messages use `role="alert"` for immediate screen reader announcement on appearance |
| Focus management | After form submission error, focus moves to error banner. After form success, focus moves to success message container (via `tabIndex={-1}` + `ref.focus()`) |
| Touch targets | All inputs, selects, buttons minimum 44px height (per Phase 2 convention) |
| Focus rings | 2px accent ring with 2px offset in dominant color on all focusable form elements (matching Phase 2 pattern) |
| Keyboard navigation | Tab through all form fields in order. Enter submits form. Escape has no special behavior (no modal to close) |
| Color independence | Error state indicated by both red border AND error text message (not color alone) |
| Reduced motion | AnimatePresence animations respect `prefers-reduced-motion` via globals.css media query. Elements appear/disappear instantly when motion is reduced. |
| Honeypot hiding | `aria-hidden="true"` + `tabIndex={-1}` ensures assistive tech users never encounter the honeypot |

## Animation Contract

All animations use the Motion library (motion/react). All respect the existing `prefers-reduced-motion` media query in globals.css.

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Field error appear | Fade in (0 to 1 opacity) + slide down (y: -4px to 0) | 200ms | easeOut |
| Field error disappear | Fade out (1 to 0 opacity) | 150ms | easeIn |
| Form exit (on success) | Fade out (1 to 0 opacity) | 150ms | easeOut |
| Success message enter | Fade in (0 to 1) + scale (0.95 to 1.0) | 300ms | easeOut |
| Error banner enter | Slide down (y: -16px to 0) + fade in (0 to 1) | 300ms | easeOut |
| Error banner exit | Fade out (1 to 0 opacity) | 150ms | easeIn |
| Submit button loading | Loader2 icon continuous rotation (360deg, infinite, linear) | 1000ms per rotation | linear |
| Scroll to form | Native `scrollIntoView({ behavior: "smooth" })` | Browser default | Browser default |

Source: Phase 2 02-CONTEXT.md D-16 (subtle & polished), D-18 (respect prefers-reduced-motion), globals.css animation tokens
