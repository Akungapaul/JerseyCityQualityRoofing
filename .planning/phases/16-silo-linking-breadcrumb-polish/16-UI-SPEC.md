## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, card content gaps (mt-2, mt-3) |
| lg | 24px | Section internal gaps (mb-6), card internal padding (p-6) |
| xl | 32px | Card internal padding on featured card (p-8) |
| 2xl | 48px | Section vertical padding (mobile: py-12) |
| 3xl | 64px | Section vertical padding (desktop: py-20) |

Exceptions: Touch targets use min-h-[44px] (44px) per WCAG 2.5.8 for all interactive card links. This is an established project convention visible in BlogCard, GuideCard, MaterialServiceCTA, and city link pills on service pages.

Source: SectionWrapper py-12/py-16/py-20 pattern, BlogCard p-6, MaterialServiceCTA p-8, existing globals.css.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 18px (1.125rem) | 500 (medium) | 1.7 | Cormorant Garamond |
| Label | 14px (0.875rem) | 700 (bold) | 1.5 | Cormorant Garamond |
| Heading | 28px (1.75rem) | 700 (bold) | 1.2 | Cormorant |
| Display | 36px (2.25rem) | 700 (bold) | 1.2 | Cormorant |

**Max sizes: 4.** No additional font sizes are introduced in this phase.

Phase 16 uses Body, Label, and Heading roles:
- **Heading** (28px/700): SiloContentLinks section heading "{serviceName} Resources & Guides"
- **Body** (18px/500): Narrative paragraph introducing the related content, breadcrumb segment text
- **Body bold variant** (18px/700): "Related Articles" sub-heading (h3). Uses the Body size (18px) at bold weight to create visual hierarchy beneath the 28px section heading without introducing a 5th font size. The weight-only distinction is sufficient because margin-top separation (mt-8) and the heading font family (Cormorant) provide additional differentiation.
- **Label** (14px/700): BlogCard silo badge text (uppercase), BlogCard metadata (date, reading time)

Source: globals.css body font-size 1.125rem, RelatedArticles text-[1.75rem] for heading, BlogCard text-[0.875rem] for badge and metadata.

## Component Inventory

### New Component: SiloContentLinks

| Property | Value |
|----------|-------|
| File path | `src/components/sections/silo-content-links.tsx` |
| Type | Server Component (no "use client") |
| Pattern source | RelatedArticles + RelatedGuides (src/components/sections/related-articles.tsx, related-guides.tsx) |
| Semantic element | `<div>` within a `<section>` (SectionWrapper provides the section) |

**Props interface:**

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| articles | InternalLink[] | yes | Blog articles from getSiloArticles() or getBlogArticlesForService() |
| costGuide | InternalLink or null | yes | Cost guide from getCostGuideForService() |
| serviceName | string | yes | Display name of the service (e.g. "Roof Repair") |

**Empty state:** Component returns null when articles.length === 0 AND costGuide is null. No empty state UI rendered. In practice, every service has at least a cost guide, so the section always renders.

**Visual specification -- Section heading and narrative:**

- Heading: `text-[1.75rem] font-heading font-bold text-text-primary mb-4` -- reads "{serviceName} Resources & Guides"
- Narrative paragraph: `text-lg text-text-secondary mb-8 leading-relaxed` -- reads "Explore our in-depth articles and cost information for {serviceName.toLowerCase()} to help you make informed decisions about your roofing project."

**Visual specification -- Cost guide featured card (when costGuide is not null):**

- Container: `bg-secondary rounded-lg p-6 border border-accent mb-8` (accent border distinguishes from blog cards)
- Inner layout: `flex items-start gap-4`
- Icon: DollarSign from Lucide React, `size={24} className="text-accent shrink-0 mt-1"` with `aria-hidden="true"`
- Content wrapper: `flex-1 min-w-0`
- Title: `text-lg font-heading font-bold text-text-primary` -- dynamic from costGuide.title
- Description: `text-lg text-text-secondary mt-1 line-clamp-2` -- dynamic from costGuide.description
- Link label: `inline-flex items-center gap-1 text-accent font-bold text-lg mt-3 group-hover:gap-2 transition-all duration-[150ms]` -- reads "View Cost Guide"
- ArrowRight icon (16px) inside link label, `aria-hidden="true"`
- Entire card is wrapped in a `<Link>` with `prefetch={false}` and `className="group block"`

**Visual specification -- Blog article cards grid (when articles.length > 0):**

- Sub-heading above grid: `text-lg font-heading font-bold text-text-primary mb-4` -- reads "Related Articles". This uses the Body size (18px) at bold weight with the heading font family, creating a weight-only sub-heading variant that stays within the 4-size typography scale.
- Grid container: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`
- Each card: Reuse existing `BlogCard` component directly (same pattern as RelatedArticles)
- Maximum 3 articles displayed (articles.slice(0, 3))
- BlogCard props: slug extracted from article.path last segment, title from article.title, description from article.description, siloBadgeLabel="ROOFING", publishDate="" (not available from InternalLink), readingTimeMinutes=10 (default estimate matching RelatedArticles pattern)

**Accessibility:**
- All cards are semantic `<Link>` (next/link) elements with visible text
- Cost guide card uses DollarSign icon with `aria-hidden="true"` to prevent screen reader duplication
- BlogCard ArrowRight icons use `aria-hidden="true"` (already implemented in BlogCard)
- Touch targets meet 44px minimum height via card padding (p-6 = 24px top + content + 24px bottom exceeds 44px)
- Focus ring inherited from default browser focus ring on Link elements
- "Related Articles" sub-heading uses h3 (not h2) since the section h2 is "Resources & Guides"

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Cost guide card hover | Border transitions from #4a5040 to #c89640 over 150ms (hover:border-accent transition-colors duration-[150ms]) |
| Cost guide card click | Navigates to cost guide page via next/link client-side navigation |
| Blog card hover | Border transitions from #4a5040 to #c89640 over 150ms (existing BlogCard behavior) |
| Blog card click | Navigates to blog article page via next/link client-side navigation |
| Blog card "Read Article" arrow | Gap expands from gap-1 to gap-2 on hover over 150ms (existing BlogCard behavior) |
| ScrollReveal | Fade-in on scroll intersection (existing ScrollReveal wrapper, no new animation logic) |
| Keyboard navigation | Tab focuses each card link sequentially. Enter activates. Default browser focus ring. |
| Breadcrumb links | Hover transitions text from text-secondary to text-accent over --duration-fast (150ms) (existing behavior, no change) |

**Responsive behavior:**
- Mobile (< 640px): Full-width cards, blog grid stacks to single column. Cost guide card fills container width.
- Tablet (640px-768px): Blog grid becomes 2 columns (md:grid-cols-2). Cost guide card fills container width.
- Desktop (> 1024px): Blog grid becomes 3 columns (lg:grid-cols-3). Cost guide card fills container width (full-width featured card is intentional -- matches MaterialServiceCTA pattern for emphasis).

No breakpoint-specific overrides needed beyond the grid responsive classes.
