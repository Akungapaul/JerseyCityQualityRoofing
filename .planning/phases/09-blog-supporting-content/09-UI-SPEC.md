## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge pill `py-1`, reading-time badge gap |
| sm | 8px | Compact element spacing, tag chips gap, TOC item padding |
| md | 16px | Default element spacing, section inner padding, card grid gaps, article body paragraph margin |
| lg | 24px | Section padding `p-6`, card padding, gap between major elements, blog sidebar spacing |
| xl | 32px | Layout column gaps (`lg:gap-8`), blog two-column gutter |
| 2xl | 48px | Section vertical padding (mapped to `py-12`) |
| 3xl | 64px | Section vertical padding on large screens (mapped to `py-16` / `py-20`) |

Exceptions:
- Touch targets: all interactive elements use `min-h-[44px] min-w-[44px]` per WCAG 2.2 Target Size
- Hero section vertical padding: `py-16 sm:py-20 lg:py-24` (64px / 80px / 96px) -- inherited from ServiceHero and CityHubHero patterns
- Max content width: 1280px (`max-w-[1280px]`) with responsive horizontal padding `px-4 sm:px-6 lg:px-8`
- Blog article body: max prose width 768px (`max-w-3xl`) within the content column for optimal line length (~65-75 characters per line at 18px)
- Table of Contents sidebar: fixed width 280px (`w-[280px]`) on desktop, full width on mobile

Source: SectionWrapper `py-12 sm:py-16 lg:py-20`, ServiceHero `py-16 sm:py-20 lg:py-24`, established Phase 2/5/7/8 patterns

## Typography

| Role | Size | Weight | Line Height | CSS Class |
|------|------|--------|-------------|-----------|
| Body | 18px (1.125rem) | 500 (medium) | 1.7 | default body styles from globals.css |
| Body Large | 18px (text-lg) | 500 | 1.625 (leading-relaxed) | `text-lg text-text-secondary leading-relaxed` |
| Label / Badge / Tag | 14px (0.875rem) | 700 (bold) | 1.5 | `text-[0.875rem] font-bold uppercase tracking-wider` |
| Article Meta | 16px (1rem) | 500 | 1.5 | `text-[1rem] text-text-secondary` |
| Section Heading (H2) | 28px (1.75rem) | 700 (bold) | 1.2 | `text-[1.75rem] font-heading font-bold text-text-primary` |
| Sub-section Heading (H3) | 22px (1.375rem) | 700 (bold) | 1.2 | `text-[1.375rem] font-heading font-bold text-text-primary` |
| Page Title (H1) | 40px/48px (2.5rem/3rem) | 700 (bold) | 1.1 | `text-[2.5rem] lg:text-[3rem] font-heading font-bold leading-[1.1]` |
| TOC Link | 16px (1rem) | 500 (medium) | 1.4 | `text-[1rem] text-text-secondary hover:text-accent transition-colors` |

Font loading: `next/font/google` self-hosts Cormorant and Cormorant Garamond. CSS variables `--font-heading` and `--font-body` set in `@theme` block.

Note: The Article Meta role (16px) is introduced for date, reading time, and author attribution metadata displayed below blog headlines. This is the only new size -- all other sizes are inherited from previous phases.

Source: globals.css base styles, ServiceHero H1 pattern, ServiceContentSection heading patterns from Phases 5/6/7/8

## Interaction Contracts

### Scroll-Linked Animations
All sections (except hero, BadgeStrip, MidPageCTA, QuoteForm, CTABanner) wrapped in `<ScrollReveal>` for fade-in-on-scroll using Motion library. Respects `prefers-reduced-motion: reduce` via globals.css media query.

### FAQ Accordion
- `"use client"` component using `useState` + `AnimatePresence` (existing Phase 4 component)
- First FAQ open by default (`defaultOpenIndex={0}`)
- Single-expand behavior (opening one closes others)
- ChevronDown icon rotates 180deg on expand
- Expand/collapse animation: height 0 to auto, opacity 0 to 1, duration 300ms, ease "easeOut"
- ARIA: `aria-expanded`, `aria-controls`, `role="region"`, `aria-labelledby`

### Table of Contents Links
- Smooth scroll to heading via `<a href="#heading-id">` (browser-native smooth scroll via `scroll-behavior: smooth` if added, or instant jump)
- Hover: `text-text-secondary` -> `text-accent` transition 150ms
- No sticky TOC behavior in Phase 9 (deferred to Phase 10 if needed)

### Blog Card Hover
- Card border: `border-[#4a5040]` -> `border-accent` on hover, 150ms transition
- Arrow icon: `gap-1` -> `gap-2` on group hover, creating a slide-right micro-animation (matching CityCard pattern)

### Guide Card Hover
- Same hover pattern as BlogCard

### Internal Navigation Links
- All internal links from the automated linking system use `next/link` with `prefetch={false}` (matching Phase 8 SiblingCitiesNav pattern for large link sets)
- Hover state: `hover:text-accent transition-colors duration-[150ms]`

### Cost Table
- Horizontal scroll on mobile via `overflow-x-auto` wrapper
- No interactive sorting or filtering (static table, content-driven)

### Material Comparison
- Static pros/cons display (no expand/collapse -- content is always visible)

## Accessibility Checklist

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<article>` wraps blog posts. `<section>` for all other page sections. `<nav>` for TOC and internal link sections. `<table>` with `<caption>`, `<thead>`, `<tbody>` for cost tables. `<ol>` for ordered identification signs. |
| Single H1 | Hero component renders the only `<h1>` per page |
| Heading hierarchy | H1 (hero) > H2 (section headings, article body H2) > H3 (article body H3, card headings, city names). No skipping levels. Article sections constrained to `headingLevel: 2 | 3` per RESEARCH.md Pitfall 6. |
| Keyboard navigation | All links, buttons, accordion triggers reachable via Tab; `focus-visible` ring on all |
| Focus ring | `box-shadow: 0 0 0 2px var(--color-dominant), 0 0 0 4px var(--color-accent)` via `.focus-ring` class. FAQ accordion uses `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-dominant`. |
| Touch targets | `min-h-[44px] min-w-[44px]` on all interactive elements including TOC links, card links, CTA buttons |
| Color contrast | Text primary #f0ede6 on dominant #2a2e22 = 10.7:1 (AAA). Text secondary #b0ae9e on dominant #2a2e22 = 5.8:1 (AA). Accent #c89640 on dominant #2a2e22 = 4.5:1 (AA). Success #34d399 on secondary #3e4435 = 4.8:1 (AA). Destructive #ef4444 on secondary #3e4435 = 4.6:1 (AA). |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all animations in globals.css |
| ARIA patterns | FAQ accordion: `aria-expanded`, `aria-controls`, `role="region"`, `aria-labelledby`. TOC: `<nav aria-label="Table of contents">`. Cost table: `<caption>` for screen readers. |
| Image alt text | No inline images in Phase 9 (text-only content pages). OG images generated via route handler. If hero images added later, require descriptive alt text per SEO-12. |
| Screen reader | TOC uses `<nav aria-label="Table of contents">`. Blog hero silo badge serves as visible category label. Ordered lists use semantic `<ol>` for identification signs. |

Source: globals.css `.focus-ring` class, WCAG AA requirements (UX-05, UX-06), established patterns in FaqAccordion and Phase 8 accessibility checklist
