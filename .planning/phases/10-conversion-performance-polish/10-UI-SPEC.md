## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, dismiss button padding |
| sm | 8px | Compact element spacing, filter bar gaps, banner internal vertical padding, pill vertical padding |
| md | 16px | Default element spacing, gallery card padding, popup inner padding, banner horizontal padding |
| lg | 24px | Gallery card `p-6`, popup content padding, filter bar section padding |
| xl | 32px | Gallery grid gap (`gap-8`), popup outer margin |
| 2xl | 48px | Section vertical padding (mapped to `py-12`) |
| 3xl | 64px | Section vertical padding on large screens (mapped to `py-16` / `py-20`) |

Exceptions:
- Touch targets: all interactive elements use `min-h-[44px] min-w-[44px]` per WCAG 2.2 Target Size
- Floating CTA: positioned `bottom-6 right-6` (24px from edges) to avoid content overlap
- Exit popup: centered with `max-w-lg` (512px), `p-8` (32px) internal padding
- Gallery comparison slider container: `aspect-[4/3]` with no explicit padding (image fills container)

Source: SectionWrapper established patterns, RESEARCH.md component specifications. Revision note: urgency banner changed from `py-3` (12px) to `py-2` (8px) and pill components changed from `py-1` (4px, already in scale) -- all values now strictly within the declared scale.

### New Hooks for Phase 10

#### useScrollPastFold

| Property | Value |
|----------|-------|
| File | `src/hooks/use-scroll-past-fold.ts` |
| Parameters | `threshold: number` (default `1.0` -- one full viewport height) |
| Returns | `boolean` -- true when `window.scrollY > window.innerHeight * threshold` |
| Listener | `window.addEventListener("scroll", handler, { passive: true })` |
| Cleanup | Removes listener on unmount |

#### useExitIntent

| Property | Value |
|----------|-------|
| File | `src/hooks/use-exit-intent.ts` |
| Parameters | `{ enabled?: boolean; delayMs?: number }` (defaults: `true`, `5000`) |
| Returns | `{ isTriggered: boolean; dismiss: () => void }` |
| Desktop detection | `document.addEventListener("mouseleave")` -- triggers when `e.clientY <= 0` and dwell time > delayMs |
| Mobile detection | `window.addEventListener("scroll", { passive: true })` -- triggers when scroll delta > 300px upward and dwell time > delayMs |
| Session persistence | Module-level `let dismissed = false` flag; once `dismiss()` called, `isTriggered` never returns true again in the session |

## Interaction Contracts

### Floating CTA Behavior

| State | Behavior |
|-------|----------|
| Page load | Hidden (user has not scrolled past fold) |
| Scroll past 1x viewport height | Animates in from bottom-right (opacity + translateY, 300ms) |
| User clicks CTA | Smooth scroll to `#quote-form` on current page; if no `#quote-form`, navigate to `/contact` |
| User clicks dismiss (X) | Animates out (300ms), remains hidden for rest of page session |
| User navigates to new page | Dismiss state resets, CTA reappears after scroll threshold |
| `#quote-form` enters viewport | CTA hides automatically (no duplicate CTA near form) |
| `#quote-form` leaves viewport | CTA reappears if not previously dismissed |
| Reduced motion | No animation, instant show/hide via CSS `@media (prefers-reduced-motion: reduce)` |

### Exit-Intent Popup Behavior

| State | Behavior |
|-------|----------|
| Page load | Hook inactive for first 5 seconds (delay gate) |
| Desktop: mouse leaves viewport top | Popup animates in (overlay fade 200ms, dialog scale+fade 300ms) |
| Mobile: rapid scroll up (>300px delta) | Same popup animation |
| User clicks CTA | Navigate to `/contact` or smooth scroll to `#quote-form` |
| User clicks close (X) or presses Escape | Popup animates out (300ms), module-level dismissed flag set |
| User clicks overlay (outside dialog) | Same as close |
| After dismissal on any page | Popup never shown again during browser session |
| Focus management | On open: focus moves to close button. On close: focus returns to previously focused element. Tab trapped within dialog. |
| Scroll lock | `body.style.overflow = "hidden"` while open; restored on close |
| Reduced motion | No animation, instant show/hide |

### Gallery Comparison Slider Behavior

| State | Behavior |
|-------|----------|
| Initial | Slider handle at 50% (center), "Before" visible on left, "After" on right |
| Mouse drag | Slider handle follows pointer; left image reveals proportionally |
| Touch drag | Same behavior with touch events |
| Keyboard | Left/Right arrow keys move handle by 5% increments (`keyboardIncrement={5}`) |
| Tab focus | Handle receives focus ring when tabbed to |
| Image loading | `aspect-[4/3]` container with `bg-[#33382b]` background shown until image loads (CLS prevention) |

### Gallery Filtering Behavior

| State | Behavior |
|-------|----------|
| No filters | All projects displayed in grid |
| Service filter selected | Grid updates instantly (client-side), URL updates via `router.replace` with `scroll: false` |
| City filter selected | Same behavior, stacks with service filter |
| Both filters active | Projects matching BOTH filters shown |
| No results | Empty state message displayed (see copywriting) |
| Clear Filters clicked | Both selects reset to "All", URL params cleared |
| Page refresh with params | Filters pre-selected from URL search params (shareable URLs) |

### Urgency Banner Behavior

| State | Behavior |
|-------|----------|
| Emergency page | Banner always renders above hero section |
| Non-emergency page (storm season) | Banner renders in marketing layout above header content area |
| Non-emergency page (outside storm season) | Banner does not render (returns null) |
| Phone link click | Initiates phone call via `tel:` href |
| No animation | Static render, no entrance/exit animation (reduces CLS risk) |

## Layout Integration Points

### Marketing Layout Changes

The marketing layout (`src/app/(marketing)/layout.tsx`) must be updated to include:

```
<Header />
<UrgencyBanner variant="storm-season" />  <!-- conditional, server-rendered -->
<Breadcrumbs />
<main id="main-content">{children}</main>
<Footer />
<FloatingCTA />                           <!-- client component, fixed position -->
<ExitIntentPopup enabled={???} />         <!-- client component, needs route awareness -->
```

**Route awareness for ExitIntentPopup:** The popup should only render on service and location pages. Two approaches:
1. Pass `enabled` prop based on route segment (requires reading pathname in layout)
2. Render on all pages but have the hook check pathname internally

Recommendation: Render `<ExitIntentPopup />` in marketing layout without `enabled` prop. The hook internally checks `window.location.pathname` against allowed prefixes (`/services/`, `/service-areas/`). This avoids making the layout a client component.

### Emergency Page Integration

Emergency service pages and emergency service-in-city pages render `<UrgencyBanner variant="emergency" />` directly in their page template, above the hero section. This is independent of the storm-season banner in the layout.

## Core Web Vitals Contract (SEO-13)

### Targets

| Metric | Target | Measurement |
|--------|--------|-------------|
| LCP (Largest Contentful Paint) | < 2.5s on mobile | Lighthouse on heaviest page (service-in-city with 15 sections) |
| INP (Interaction to Next Paint) | < 200ms | All interactive elements (accordion, carousel, forms, sliders) |
| CLS (Cumulative Layout Shift) | < 0.1 | All pages, particularly gallery page with image loading |

Note: Requirements say "FID < 100ms" but FID was replaced by INP on March 12, 2024. Target INP < 200ms per Google's "good" threshold.

### LCP Optimization Rules

| Rule | Implementation |
|------|----------------|
| Hero images | Use `preload` prop (not deprecated `priority`) on above-fold images in Next.js 16 |
| Font loading | Verify `font-display: swap` is applied via `next/font/google`. No `@font-face` overrides with `font-display: block`. |
| Gallery images | ALL lazy-loaded (`loading="lazy"` default in `next/image`). Only first visible card pair gets `preload` if above fold. |
| Third-party scripts | Turnstile script deferred. No render-blocking external scripts. |

### CLS Prevention Rules

| Rule | Implementation |
|------|----------------|
| Comparison slider container | `aspect-[4/3]` with `bg-[#33382b]` background placeholder |
| Floating CTA | `position: fixed` -- never causes layout shift |
| Urgency banner | Server-rendered with fixed content height -- no shift on hydration |
| Images | All `next/image` usages require explicit `width` and `height` or `fill` with `sizes` attribute |
| Fonts | `font-display: swap` with appropriate `size-adjust` from `next/font` |

### INP Optimization Rules

| Rule | Implementation |
|------|----------------|
| Event handlers | All scroll listeners use `{ passive: true }` |
| Gallery filtering | Client-side array filter, no server round-trip |
| Accordion toggle | Single `useState` update, no complex re-renders |
| Exit popup open/close | Animation via Motion (GPU-accelerated), minimal main thread work |

## Image Alt Text Audit Contract (SEO-12)

### Audit Rules

| Rule | Check |
|------|-------|
| No `<img>` without `next/image` | Grep for `<img` in all `.tsx` files; only allowed inside third-party components (react-compare-slider) |
| No empty alt text on content images | Every `alt` must be descriptive, not a filename, not empty (unless decorative with `alt=""`) |
| Gallery image alt pattern | "[Condition] [roof type] on [building type] in [neighborhood], [city]" -- e.g., "Damaged asphalt shingle roof on brownstone in Jersey City Heights" |
| Decorative images | SVG illustrations, icons with `aria-hidden="true"` may use `alt=""` or omit alt |
| Placeholder images | Alt text describes what the final image WILL show, not "placeholder" |

## Responsive Breakpoints

All breakpoints use Tailwind CSS 4 defaults:

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Default (mobile) | < 640px | Single-column gallery grid, floating CTA icon-only, exit popup full-width minus margin |
| sm | >= 640px | Floating CTA shows text label, filter bar inline layout |
| md | >= 768px | Gallery grid 2-column |
| lg | >= 1024px | Gallery grid 2-column (capped at 2 for comparison usability), exit popup max-w-lg centered |

Max content width: `max-w-[1280px]` with `px-4 sm:px-6 lg:px-8` horizontal padding.

## Accessibility Checklist

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<section>`, `<nav>`, `<dialog>` semantics; gallery cards in `<article>` wrappers |
| Single H1 per page | Gallery page: GalleryHero renders the only `<h1>` |
| Heading hierarchy | H1 (gallery hero) > H2 (section headings, popup heading) > H3 (card titles) |
| Keyboard navigation | Gallery slider: arrow keys. Popup: Tab trap, Escape to close. Floating CTA: tabbable. Filter selects: native keyboard support. |
| Focus ring | `box-shadow: 0 0 0 2px var(--color-dominant), 0 0 0 4px var(--color-accent)` via `focus-visible` classes |
| Touch targets | `min-h-[44px] min-w-[44px]` on all buttons, links, select elements |
| Color contrast | Text primary #f0ede6 on dominant #2a2e22 = 10.7:1 (AAA). Text dominant #2a2e22 on emergency accent #d4782f = 4.7:1 (AA). Accent #c89640 on dominant #2a2e22 = 4.5:1 (AA). |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all animations per globals.css |
| ARIA patterns | Popup: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`. Banner: `role="alert"`. Floating CTA: `role="complementary"`, `aria-label`. Filters: `aria-label` on each select. |
| Screen reader | Before/After labels visible as text overlay. Popup announced via dialog role. Banner announced via alert role. |
| Scroll lock | Body scroll prevented while exit popup is open |

Source: globals.css `.focus-ring` class, WCAG AA requirements (UX-05, UX-06), established accessibility patterns
