    **Image Audit (SEO-12):**

    1. Run comprehensive grep for raw `<img` tags across all .tsx files:
       ```bash
       grep -rn '<img ' src/ --include='*.tsx' --include='*.ts'
       ```
       Expected: ZERO results (all images should use `next/image` or `ReactCompareSliderImage`).
       If any raw `<img` tags found, replace with `next/image` component with proper `width`, `height`, and descriptive `alt`.

    2. Run grep for empty or filename-based alt text:
       ```bash
       grep -rn 'alt=""' src/ --include='*.tsx' | grep -v 'aria-hidden'
       grep -rn 'alt=".*\.\(jpg\|png\|webp\|svg\)"' src/ --include='*.tsx'
       ```
       - `alt=""` is allowed ONLY on decorative images that also have `aria-hidden="true"`
       - Alt text containing file extensions (.jpg, .png, .webp, .svg) must be replaced with descriptive text
       - Fix any violations found

    3. Verify all `next/image` usages have explicit dimensions or `fill` prop:
       ```bash
       grep -rn 'next/image' src/ --include='*.tsx'
       ```
       Check each usage has `width`/`height` or `fill` with `sizes` attribute.

    **CWV Optimization (SEO-13):**

    4. Verify font loading is correct:
       ```bash
       grep -rn 'font-display' src/ --include='*.css' --include='*.tsx' --include='*.ts'
       ```
       Must NOT contain `font-display: block`. The `next/font/google` in `src/app/layout.tsx` should use `display: "swap"`. Verify this is set correctly.

    5. Verify all scroll listeners use passive:
       ```bash
       grep -rn 'addEventListener.*scroll' src/ --include='*.ts' --include='*.tsx'
       ```
       Every scroll listener must include `{ passive: true }`. Fix any that don't.

    6. Verify no render-blocking external scripts:
       ```bash
       grep -rn '<script' src/ --include='*.tsx' | grep -v 'async\|defer\|next/script'
       ```
       Should return zero results. Any `<script>` tags must use `next/script` with `strategy="afterInteractive"` or `strategy="lazyOnload"`.

    7. Verify CLS prevention on gallery cards:
       - `src/components/sections/gallery-comparison-card.tsx` must have `aspect-[4/3]` container with `bg-[#33382b]` background
       - Confirm these classes exist in the file

    8. Check for preload on hero images:
       ```bash
       grep -rn 'preload\|priority\|fetchPriority' src/ --include='*.tsx'
       ```
       - Per RESEARCH.md: Next.js 16 uses `preload` instead of deprecated `priority`
       - Above-fold hero images should have `preload` prop
       - If any use `priority`, note this for awareness (it still works but is deprecated)

    **Build Verification:**

    9. Run `pnpm test && pnpm build` to confirm no regressions from any fixes applied.
