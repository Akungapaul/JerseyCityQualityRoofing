## Non-Visual Fixes (Code-Only)

All three fixes in this phase have zero visual impact. Documented here for downstream consumer completeness.

### Fix 1: Blog Silo Data Correction (CONT-01)

- **File:** `src/data/content/blog/preventative-roof-maintenance-checklist.ts`
- **Change:** Update `siloCategory` from `'residential'` to `'commercial'` and `parentPillarLink` from `'/services/residential/roof-maintenance'` to `'/services/commercial/roof-maintenance'`
- **Visual impact:** None. The SiloPillarLink component on the blog article page will link to a valid route instead of a 404. The link text and styling remain identical.

### Fix 2: Content Registry Node Registration (SEO-05)

- **File:** `src/lib/internal-links.ts`
- **Change:** Add registration loops for 8 service nodes and 12 city nodes inside `initializeContentRegistry()`
- **Visual impact:** None directly. The `ProblemSolutionCTA` component on problem pages (`/problems/[slug]`) will render service links where it previously rendered zero. These links use the existing Link styling (accent color, inherited typography). No new visual patterns.

### Fix 3: FloatingCTA Contact Page ID Alignment (CRO-03)

- **File:** `src/app/(marketing)/contact/page.tsx`
- **Change:** Add `id="quote-form"` to a wrapper element around the QuoteForm section, matching the pattern used on 13+ other pages
- **Visual impact:** None. The FloatingCTA button will scroll to the form section on the contact page instead of self-linking to `/contact`. The button appearance and behavior are unchanged -- only the anchor target resolves correctly.

## Interaction Contracts

No new interactions are introduced. All existing interactions carry forward unchanged.

The only behavioral change is FloatingCTA on the `/contact` page:

| Property | Before Fix | After Fix |
|----------|-----------|-----------|
| FloatingCTA click on /contact | Navigates to `/contact` (self-link, no-op) | Smooth scrolls to `#quote-form` anchor on same page |
| IntersectionObserver on /contact | Finds no element, falls back to href="/contact" | Observes `id="quote-form"` element, shows/hides based on form visibility |

This corrects existing behavior to match the established pattern on all other pages. No new animation, timing, or interaction design is needed.

## Accessibility Contract

No new accessibility requirements. The `id="quote-form"` wrapper on the contact page improves accessibility by enabling the FloatingCTA to function as a same-page anchor link (preferable to a full page navigation for screen reader users who are already on the contact page).
