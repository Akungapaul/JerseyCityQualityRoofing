**Color constraints per locked decisions (MUST be honored):**
- D-01: Dark and authoritative mood
- D-02: Start fresh -- NO navy (#1a365d) or burnt orange (#c05621). Entirely new palettes.
- D-03: Deep-toned backgrounds -- deep blues, slates, dark greens, dark warm grays. NOT near-black/charcoal (#1a1a1a, #111, #0d0d0d are forbidden).
- D-04: Full dark site -- dark backgrounds on ALL sections. Light text on dark. Slightly lighter dark tones for depth between sections.
- D-05: Warm accent colors -- gold, amber, copper, warm orange family. Strong contrast on dark backgrounds.
- D-06: Modern-elegant typographic treatment -- clean spacing, lighter weights, generous whitespace
- D-07: Flat backgrounds -- depth via shadows/elevation only. No textures, noise, grain, gradients.

**Color role assignments each variation must include (from UI-SPEC):**
- Dominant surface (60%): The main dark background. Deep-toned, not charcoal.
- Secondary surface (30%): Slightly lighter/shifted dark tone. Cards, dropdowns, footer columns, alternating section bands.
- Accent (10%): Warm (gold/amber/copper/warm-orange). ONLY used on: CTA buttons, phone links, active nav indicators, focus rings, hover underlines, star ratings.
- Text primary: Light/white tone. Minimum 4.5:1 contrast ratio on dominant surface.
- Text secondary: Muted light tone. Minimum 4.5:1 contrast ratio on dominant surface.
- Destructive: Red-toned for errors (form validation).
- Success: Green-toned for confirmations.

**10 variations should explore different dominant surface directions:**
1-2: Deep slate blue family (e.g., #1e293b, #1e3a5f range)
3-4: Deep forest green / teal family (e.g., #1a2e2a, #1c3d3a range)
5-6: Deep warm gray / charcoal-brown family (e.g., #2d2926, #3b3430 range)
7-8: Deep navy-slate hybrid (e.g., #1a2744, #1e2d4a range)
9-10: Mixed/creative -- one using deep plum/wine, one using deep olive/sage

Each file should include inline CSS with the Google Fonts import for Cormorant and Cormorant Garamond. Each file must show contrast ratios as text annotations next to color swatches.

File naming: `color-variations/palette-01.html` through `color-variations/palette-10.html`

**Step 2: Create `src/components/ui/button.tsx`**

Build a Button component using cva (class-variance-authority) with these exact variants per UI-SPEC:

```typescript
"use client";

import { cva, type VariantProps } from "class-variance-authority";
```

Variant definitions:
- `variant`:
  - `primary`: Accent background color, dominant text color. For "Get Free Quote" CTA buttons. Hover: slightly lighter accent.
  - `secondary`: Transparent background, accent border, accent text. For secondary actions on dark backgrounds. Hover: accent bg with dominant text.
  - `phone`: Accent text color, no background, underline on hover. Wraps around tel: link content. For phone number display.
- `size`:
  - `default`: px-6 py-3, text-lg (18px), min-h-[44px] (touch target)
  - `compact`: px-4 py-2, text-lg (18px), min-h-[44px] (touch target)

All variants MUST include:
- `min-h-[44px] min-w-[44px]` (WCAG 2.2 touch target)
- `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-dominant` (focus ring per UX-05)
- `font-body font-medium` (Cormorant Garamond 500)
- `transition-colors` with `duration-fast` (150ms)
- `inline-flex items-center justify-center` (layout)
- `rounded-md` (subtle rounding)
- `disabled:opacity-50 disabled:pointer-events-none` (disabled state)

The component must accept standard button HTML attributes plus an `asChild` boolean prop (when true, renders as a Slot/child element for composing with Next.js Link). If asChild is complex, keep it simple: just use a standard `<button>` element but also export the cva styles so layout components can apply them to `<a>` or `<Link>` elements directly.
