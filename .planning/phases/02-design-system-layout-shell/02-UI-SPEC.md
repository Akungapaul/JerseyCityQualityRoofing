# Phase 02 -- UI Design Contract

> Visual and interaction contract for the design system and layout shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind 4 + cva) |
| Preset | not applicable |
| Component library | none (custom primitives with cva + cn()) |
| Icon library | Lucide React (0.477.x) |
| Font heading | Cormorant, serif (weights: 500, 700) |
| Font body | Cormorant Garamond, serif (weights: 500, 700) |
| Animation | Motion library (12.x) |
| Utility | cn() via clsx + tailwind-merge (already in src/lib/utils.ts) |

**Rationale for no shadcn:** The project uses a full dark custom theme with industry-specific styling. Cormorant serif typography diverges from shadcn defaults significantly. Custom primitives with cva provide the same variant pattern without overriding shadcn defaults on every component. The cn() utility and cva are already available or planned in the stack.

## Color

**IMPORTANT:** The specific hex values below are placeholder references defining the color ROLES and CONSTRAINTS. Per CLAUDE.md hard requirement (UX-02), 10 color theme variations must be generated as standalone HTML files for user approval BEFORE implementation. The approved palette replaces these role assignments.

### Color Role Contract

| Role | Constraint | Usage |
|------|------------|-------|
| Dominant surface (60%) | Deep-toned dark: deep blue, slate, dark green, or dark warm gray. NOT near-black/charcoal. | Page background, all content section backgrounds |
| Secondary surface (30%) | Slightly lighter or shifted dark tone from dominant. Provides depth without breaking dark theme. | Cards, navigation dropdowns, mega-menu panels, footer columns, section alternation bands |
| Accent (10%) | Warm family: gold, amber, copper, or warm orange. Must pass WCAG AA contrast on both dominant and secondary surfaces. | Reserved ONLY for: CTA buttons, phone number links, active nav indicators, focus rings, hover underlines, star ratings |
| Text primary | Light/white tone. Minimum 4.5:1 contrast on dominant surface. | All body text, headings, labels |
| Text secondary | Muted light tone. Minimum 4.5:1 contrast on dominant surface. | Supporting text, meta information, breadcrumb separators, footer secondary links |
| Destructive | Red-toned, visible on dark backgrounds. | Error states only (form validation). No destructive user actions exist in this phase. |
| Success | Green-toned, visible on dark backgrounds. | Form submission confirmation (Phase 3 forward) |

### Color Application Rules

- **Full dark site (D-04):** Every section uses dark backgrounds. No white or light content areas.
- **Depth via tone variation (D-07, D-15):** Alternating sections on long pages use dominant vs. secondary surface to create visual rhythm and break up 3000+ word pages.
- **Flat backgrounds only (D-07):** No textures, noise, grain patterns, or gradients. Depth via box-shadow/elevation and tone shifts.
- **Accent is restricted:** Never use accent as a background fill for content areas. Accent appears only on the elements listed above.

### 10-Variation Generation Contract

Before any color implementation, the executor MUST:
1. Generate 10 standalone HTML files, each demonstrating a complete palette applied to: header (full + shrunk), hero section with CTA, body text section, card component, mega-menu panel, footer with CTA banner, and breadcrumbs
2. Each variation explores a different combination within the constraints (D-01 through D-07)
3. Present all 10 to the user for approval
4. Selected palette hex values are recorded back into this section and globals.css @theme block is updated

## Interaction Contracts

### Sticky Header Shrink (D-09)

| Property | Full State | Shrunk State | Transition |
|----------|-----------|--------------|------------|
| Height | 80px | 56px | 300ms ease-out via Motion |
| Logo | Full size | Reduced ~75% | Scale transform |
| Navigation | Full text links | Same (desktop), hamburger only (mobile -- no change) | n/a |
| Phone number | Visible | Visible | No change |
| CTA button | Full "Get Free Quote" | Compact "Free Quote" | Width transition |
| Trigger | scroll position = 0 | scroll position > 100px | useScroll threshold |

### Mega-Menu (D-08)

| Trigger | Behavior |
|---------|----------|
| Desktop hover on nav item | Panel slides down (200ms ease-out). 100ms delay before close on mouse-leave. |
| Desktop keyboard (Enter/Space) | Panel opens. Tab cycles through panel links. Escape closes and returns focus to trigger. |
| Desktop click on nav item | Panel toggles open/closed. |
| Panel content | Grid layout: service icon + name + short description for each service in that silo. Locations panel shows all 12 municipalities in columns. |
| Close conditions | Mouse leaves panel area (after 100ms delay), Escape key, click outside, route change. |

### Mobile Navigation (D-10)

| Trigger | Behavior |
|---------|----------|
| Hamburger tap | Full-screen overlay slides in from right (300ms). Body scroll locks. |
| Silo section tap | Accordion expands to show child links (200ms). Only one section open at a time. |
| Phone/CTA at top | Always visible at top of overlay. Phone number + "Get Free Quote" button. |
| Close | X button, Escape key, or swipe right. |
| Focus trap | Focus stays within overlay while open. First focusable: close button. |

### Scroll Reveal Animations (D-16, D-17, D-18)

| Property | Value |
|----------|-------|
| Effect | fade-in-up (opacity 0 to 1, translateY 24px to 0) |
| Duration | 600ms |
| Easing | easeOut |
| Stagger | 100ms between sibling elements in lists/grids |
| Trigger | Element enters viewport at 20% visibility |
| Fires once | Yes -- no re-animation on scroll back up |
| Reduced motion | All animations disabled. Elements render at final position immediately. |

## Accessibility Contract

| Requirement | Implementation | Source |
|-------------|---------------|--------|
| Semantic HTML (UX-04) | `<header>`, `<nav>`, `<main>`, `<footer>`, `<section>` with descriptive aria-labels. Navigation uses `<nav aria-label="Main navigation">`. Breadcrumbs use `<nav aria-label="Breadcrumb">`. | REQUIREMENTS.md |
| Keyboard navigation (UX-05) | All interactive elements reachable via Tab. Mega-menu: Enter/Space to open, Escape to close, arrow keys within. Mobile nav: focus trap. | REQUIREMENTS.md |
| Focus-visible rings (UX-05) | 2px solid accent color, 2px offset. Applied via `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-2 focus-visible:ring-offset-dominant`. Visible on dark backgrounds. | REQUIREMENTS.md |
| WCAG AA contrast (UX-06) | All text/background combinations verified at 4.5:1 (normal text) or 3:1 (large text 18px+ bold or 24px+ regular). Accent on dark verified at 4.5:1. | REQUIREMENTS.md |
| Reduced motion (D-18) | `@media (prefers-reduced-motion: reduce)` disables all Motion animations. Elements appear at final position. Header shrink still functions but without animation. | CONTEXT.md |
| Skip navigation | Hidden link "Skip to main content" appears on focus before header. Targets `<main id="main-content">`. | Best practice |
| Touch targets | Minimum 44x44px for all tappable elements. | WCAG 2.2 |
| tel: links (UX-08) | Phone numbers wrapped in `<a href="tel:+12015550123">`. | REQUIREMENTS.md |
| next/link (UX-08) | All internal navigation uses Next.js Link component. | REQUIREMENTS.md |
