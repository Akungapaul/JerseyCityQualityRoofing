# Phase 11 -- UI Design Contract

> Visual and interaction contract for SEO & data consistency fixes. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase nature:** This is a bug-fix and consistency phase, not a visual design phase. The visual contract is narrow: three new silo index pages that reuse existing components and patterns established in Phases 2-10. The remaining fixes (JSON-LD anchor, OG route constants, sitemap entries) are code-only changes with zero visual impact.

## New Pages -- Visual Contract

Phase 11 creates three silo index pages. Each follows the established hub page pattern (blog hub, service areas hub).

### Page 1: /services (All Services Index)

**Purpose:** Root silo page listing all 8 services categorized into Residential and Commercial. Differentiates from the homepage services grid by emphasizing the category split as the primary organizational structure.

**Layout (top to bottom):**

| Section | Tone | Component | Content |
|---------|------|-----------|---------|
| 1. Hero | dominant | SectionWrapper + ScrollReveal + h1 | h1: "Our Roofing Services". Subtitle: "Complete residential and commercial roofing solutions for every property in Hudson County." Category badge: "ALL SERVICES" |
| 2. Intro prose | dominant | SectionWrapper + paragraph | 2-3 paragraphs (100-200 words) explaining residential vs. commercial service categories and how to navigate |
| 3. Residential block | secondary | SectionWrapper + ScrollReveal + h2 + card grid | h2: "Residential Roofing Services". 4 service cards (2-col mobile, 4-col desktop) linking to /services/residential/[service]. Each card: Lucide icon + service name + first sentence of shortDescription + ArrowRight |
| 4. Commercial block | dominant | SectionWrapper + ScrollReveal + h2 + card grid | h2: "Commercial Roofing Services". 4 service cards (same pattern) linking to /services/commercial/[service] |
| 5. Category CTAs | secondary | SectionWrapper + 2 link cards | Two prominent cards: "View All Residential Services" -> /services/residential, "View All Commercial Services" -> /services/commercial. Cards use accent border-left (4px) on hover. |
| 6. CTA Banner | secondary | CTABanner | Default copy: "Ready to Protect Your Roof?" |

**Service card pattern:** Reuse the exact card markup from `ServicesGrid` component (src/components/sections/services-grid.tsx lines 44-70). Cards use `bg-dominant` on secondary sections, `bg-secondary` on dominant sections, with `hover:bg-[#4a5040]` transition.

**JSON-LD:** `buildCollectionPageJsonLd` + `buildRoofingContractorJsonLd` + `buildBreadcrumbJsonLd` (3 schemas).

## Non-Visual Fixes (Code-Only)

These fixes have no visual impact but are part of the phase scope. Documented here for completeness.

### Fix 1: JSON-LD @id Anchor
- **File:** `src/lib/seo/json-ld.tsx`
- **Change:** Add `'@id': \`${BASE_URL}/#organization\`` to `buildRoofingContractorJsonLd()` return object
- **Visual impact:** None (invisible structured data)

### Fix 2: OG Route Constants
- **File:** `src/app/api/og/route.tsx`
- **Change:** Import `SITE_NAME` and `PHONE_NUMBER` from `@/lib/constants`. Replace literal "Jersey City Quality Roofing" with `SITE_NAME` and "(201) 555-0123" with `PHONE_NUMBER`
- **Visual impact:** None (OG images already display the same values; this ensures future constant changes propagate)

### Fix 3: Sitemap Entries
- **File:** `src/app/sitemap.ts`
- **Change:** Add 3 entries to staticPages array: `/services`, `/services/residential`, `/services/commercial`
- **Visual impact:** None (XML file consumed by crawlers only)
