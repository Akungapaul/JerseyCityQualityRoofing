**Each service MUST include ALL fields from the Service interface (per D-12, D-19):**

For EACH of the 8 services:
- `name`: Display name (e.g., "Roof Repair")
- `slug`: URL slug per D-01
- `category`: `'residential'` or `'commercial'`
- `shortDescription`: 1-2 sentence summary (~50 words)
- `fullDescription`: 3-4 sentence detailed description (~100 words)

**Part C: Create `src/data/service-city-content.ts`**

Per D-13: Cross-reference resolver. `getCityServiceContent(serviceSlug, citySlug)` returns city-specific content fragments that differentiate the same service in different cities.

```typescript
import type { ServiceCityContent, FAQ } from './types';
import { getMunicipality } from './municipalities';
import { getService } from './services';

export function getCityServiceContent(serviceSlug: string, citySlug: string): ServiceCityContent | null {
  const city = getMunicipality(citySlug);
  const service = getService(serviceSlug);
  if (!city || !service) return null;

  // Generate city-specific content fragments using municipality data
  // This ensures each service-in-city page has unique content
  return {
    serviceSlug,
    citySlug,
    localContext: buildLocalContext(city, service),
    specificConcerns: buildSpecificConcerns(city, service),
    localStats: buildLocalStats(city),
    neighborhoodMention: city.neighborhoods[0] || city.name,
    uniqueFaqs: buildUniqueFaqs(city, service),
  };
}
```

The helper functions (`buildLocalContext`, `buildSpecificConcerns`, `buildLocalStats`, `buildUniqueFaqs`) should dynamically compose content fragments using the city's actual data (housing stock, weather patterns, landmarks, architecture styles) combined with the service's specifics. This is NOT hardcoded per city+service combination -- it uses the structured data to generate unique fragments.
