**Step 1: Scaffold Next.js 16 project.**

The project root already has files (.planning/, CLAUDE.md, .claude/, _bmad/, etc.). Per RESEARCH.md Pitfall 2, `create-next-app` in a non-empty directory may conflict. Use the safest approach:

Run: `pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"`

If that fails due to non-empty directory, fall back to manual installation:
```bash
pnpm add next@latest react@latest react-dom@latest
pnpm add -D typescript @types/react @types/node @tailwindcss/postcss postcss eslint eslint-config-next
```
Then manually create tsconfig.json, next.config.ts, etc.

After scaffold completes, install additional Phase 1 dependencies:
```bash
pnpm add schema-dts clsx tailwind-merge
```

Add `type-check` script to package.json:
```json
"type-check": "tsc --noEmit"
```

`src/app/layout.tsx`:
- Import `Cormorant` and `Cormorant_Garamond` from `next/font/google`
- Cormorant: `weight: ['500', '700']`, `variable: '--font-heading'`, `display: 'swap'`, `subsets: ['latin']`
- Cormorant_Garamond: `weight: ['500', '700']`, `variable: '--font-body'`, `display: 'swap'`, `subsets: ['latin']`
- Per UI-SPEC: only 2 font weights (500, 700) for each family
- Set `<html lang="en" className={`${cormorant.variable} ${cormorantGaramond.variable}`}>`
