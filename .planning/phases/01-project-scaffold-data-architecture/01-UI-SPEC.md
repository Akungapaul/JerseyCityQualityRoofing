# Phase 1 --- UI Design Contract

> Visual and interaction contract for Phase 1: Project Scaffold & Data Architecture. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope note:** This phase is infrastructure-only. No user-facing pages, no design system, no forms, no components. The contract below covers ONLY the foundational seeds that Phase 1 plants: font loading in root layout, Tailwind CSS custom property stubs in globals.css, and minimal copy for error/404 pages. The full design system (colors, spacing, components, typography scale) is delivered in Phase 2.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 18px | 500 (medium) | 1.5 | Cormorant Garamond |
| Label | 14px | 600 (semibold) | 1.4 | Cormorant Garamond |
| Heading | 28px | 700 (bold) | 1.2 | Cormorant |
| Display | 40px | 700 (bold) | 1.1 | Cormorant |

**Source:** CLAUDE.md mandates Cormorant Garamond (medium weight) for body, Cormorant for headings, minimum 18px body. REQUIREMENTS.md UX-01 confirms.

**Phase 1 application:** Root layout loads both font families via `next/font/google` and sets CSS variables `--font-heading` and `--font-body`. No typography scale is applied to stub pages -- stubs render with browser defaults plus the font-family variables. The exact sizes, weights, and line-heights declared above are the canonical reference for Phase 2 implementation.

**Font loading contract (Phase 1 deliverable):**
- `Cormorant` loaded with weights: 400, 500, 600, 700 -- variable `--font-heading`
- `Cormorant_Garamond` loaded with weights: 400, 500, 600, 700 -- variable `--font-body`
- Both use `display: 'swap'` and `subsets: ['latin']`
- Root `<html>` element receives both CSS variable classes

|------|-------|-------|
| Dominant (60%) | TBD Phase 2 | Background, surfaces |

Accent reserved for: primary CTA buttons, click-to-call phone number, form submit buttons, active navigation indicators

**Source:** REQUIREMENTS.md UX-02 specifies "Dark/professional color theme appropriate for roofing industry" with "10 variations generated for approval before implementation." CLAUDE.md confirms the 10-variation requirement.

**Phase 1 application:** No colors are applied in Phase 1. The `globals.css` `@theme {}` block includes placeholder custom properties (`--color-primary` and `--color-secondary`) as stubs that Phase 2 replaces with the approved palette. Stub pages render with browser default colors (black text on white background).

**Stub custom properties in globals.css:**
```css
@theme {
  --font-heading: 'Cormorant', serif;
  --font-body: 'Cormorant Garamond', serif;
  --color-primary: #1a365d;
  --color-secondary: #c05621;
}
```

These hex values are PLACEHOLDERS ONLY, carried forward from RESEARCH.md Pattern 6. They will be replaced in Phase 2 after the 10-variation color approval process.

## Phase 1 Visual Deliverables Summary

Phase 1 is an infrastructure phase. The following is an exhaustive list of everything with visual impact:

1. **Root layout** (`src/app/layout.tsx`): Sets `<html lang="en">` with font CSS variable classes on the element. Sets `<body>` as the content wrapper. No styling beyond font variables.

2. **globals.css** (`src/styles/globals.css`): Contains `@import "tailwindcss"` and a `@theme {}` block with font family variables and placeholder color properties.

3. **not-found.tsx** (`src/app/not-found.tsx`): Renders semantic `<main>` with `<h1>` and `<p>`. No styling. Copy per Copywriting Contract above.

4. **error.tsx** (`src/app/error.tsx`): Client component. Renders `<main>` with `<h1>`, `<p>`, and a `<button>` calling `reset()`. No styling. Copy per Copywriting Contract above.

5. **Stub pages** (all route pages): Each renders `<main>` with `<h1>` containing the page name and `<p>` with "under construction" text. No styling.

6. **Marketing layout** (`src/app/(marketing)/layout.tsx`): Pass-through layout that renders `{children}`. No header, footer, or navigation (those are Phase 2).

Everything else in Phase 1 is invisible infrastructure: data registries, SEO helpers, sitemap, robots.txt, metadata generators, type definitions.
