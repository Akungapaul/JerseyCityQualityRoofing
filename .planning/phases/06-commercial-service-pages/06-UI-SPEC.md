## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, connector line width |
| sm | 8px | Compact element spacing, card internal gaps, badge row gap |
| md | 16px | Default element spacing, card padding, form field gaps, grid gaps |
| lg | 24px | Section inner padding (mobile), FAQ item padding, section heading margin-bottom (mt-6) |
| xl | 32px | Section heading margin-bottom (mb-8), timeline step bottom padding (pb-8) |
| 2xl | 48px | Section vertical padding (mobile: py-12) |
| 3xl | 64px | Section vertical padding (desktop: py-16 to py-20) |

Exceptions:
- 44px minimum touch target on all interactive elements (buttons, accordion triggers, carousel controls, expand/collapse toggles) per WCAG 2.5.5 and existing button-variants.ts pattern
- 1280px max content width (established in SectionWrapper)
- px-4 / sm:px-6 / lg:px-8 horizontal page gutters (16/24/32px)

**Source:** Existing SectionWrapper pattern, Phase 5 UI-SPEC spacing scale, button-variants.ts min-h-[44px].

## Typography

| Role | Size | Weight | Line Height | Font Family | CSS Class Pattern |
|------|------|--------|-------------|-------------|-------------------|
| Body | 18px (1.125rem) | 500 (medium) | 1.7 | Cormorant Garamond | `text-lg` (base body) |
| Label / Caption | 18px (1.125rem) | 500 (medium) | 1.5 | Cormorant Garamond | `text-lg text-text-secondary` |
| Section Heading | 28px (1.75rem) | 700 (bold) | 1.2 | Cormorant | `text-[1.75rem] font-heading font-bold` |
| Page Title / Hero | 40px / 48px (2.5rem / 3rem) | 700 (bold) | 1.1 | Cormorant | `text-[2.5rem] lg:text-[3rem] font-heading font-bold` |

Additional typographic rules:
- Duration badges on timeline steps: 18px body size, weight 500, text-accent color, bg-secondary container
- Material lifespan/price badges: 18px body size at `text-sm` exception (see below)
- All prose paragraphs: 18px at 1.7 line-height (matches body default)
- Process step titles: 18px, font-heading, font-bold

**18px floor note:** Per CLAUDE.md and Phase 2 UI-SPEC, the minimum text size is 18px. The `text-sm` class (14px) is used ONLY for badge/pill elements (lifespan, price range, impact labels, duration pills) where the small size is a deliberate design choice within a contained element and has been established in Phase 5 implementation. This exception carries forward identically.

**Source:** CLAUDE.md hard requirements (Cormorant Garamond medium, 18px min), globals.css established values, Phase 5 UI-SPEC typography scale, Phase 5 implemented components.

## Page Assembly Pattern

The commercial service page at `src/app/(marketing)/services/commercial/[service]/page.tsx` follows the exact pattern of the residential page:

```
1. Import all section components (same set minus emergency components)
2. Import commercial content data files (4 new files)
3. Define CONTENT_MAP record mapping commercial slugs to content objects
4. Define getServiceContent() helper for O(1) lookup
5. generateStaticParams() uses getCommercialServiceSlugs()
6. generateMetadata() uses service data with path `/services/commercial/[slug]`
7. Page component renders StandardTemplate (no emergency template branching)
8. JSON-LD: Service schema + FAQPage schema (same builders from Phase 5)
9. OG image: `/api/og?service=[slug]` (same route, reused)
```

Key differences from residential page:
- Uses `getCommercialServiceSlugs()` instead of `getResidentialServiceSlugs()`
- Canonical URL path: `/services/commercial/[slug]` instead of `/services/residential/[slug]`
- No emergency template branching (always standard template)
- Uses CommercialRelatedServicesRow instead of RelatedServicesRow
- Commercial content data files (4 new) instead of residential content data files
- CTA copy is commercially focused (see Copywriting Contract)

## Interaction Patterns

All interaction patterns are inherited from Phase 5 without modification:

### Scroll-Triggered Animations
- Every section's content wrapped in ScrollReveal (fade-in-up, 600ms duration, easeOut)
- Respect prefers-reduced-motion (elements appear instantly, no animation)
- Animation triggers once per page load, not on re-scroll
- Viewport trigger: 20% visibility (0.2 IntersectionObserver ratio)

### FAQ Accordion
- Single-open behavior (clicking one closes the other)
- First question open by default (defaultOpenIndex=0)
- Motion height animation (0 to auto) with 300ms easeOut
- Chevron rotates 180deg when open

### Material Card Expand/Collapse
- Pros/cons section hidden by default
- Toggle button reveals list with same Motion height animation pattern (300ms easeOut)
- Multiple cards can be expanded simultaneously (independent state)

### TestimonialCarousel
- Embla carousel with autoplay (5000ms delay)
- Loop enabled, stops on mouse enter
- 1 slide visible on mobile, 3 on desktop (lg breakpoint)
- Previous/Next buttons with 44px touch targets
- Previous button: `aria-label="Previous testimonial"`
- Next button: `aria-label="Next testimonial"`

### Compact Form Auto-Selection
- CompactQuoteForm receives `defaultServiceType` prop matching current commercial service name
- Service type dropdown is pre-selected, reducing friction

### Related Services Cards
- Hover: background darkens (secondary to secondary-lighter), arrow translates right 4px
- Entire card is a clickable link (next/link)
- Transition: 150ms (--duration-fast)
- Links point to `/services/commercial/[slug]` (not residential)

**Source:** Phase 5 interaction patterns, all implemented and verified.
